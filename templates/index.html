Sistema Transaccional para el Control de Recaudaciones y Flujo de Pasajeros en la Red Mi Teleferico

GABRIEL OMAR ANDIA ALAVE MARCOS DANIEL FERNANDEZ CAZAS

1. Introduccion

La movilidad urbana en zonas de topografia compleja presenta desafios unicos que los sistemas de transporte tradicionales no pueden resolver con eficiencia. La Red "Mi Teleferico", uniendo las ciudades de La Paz y El Alto, se ha consolidado como una solucion de vanguardia tecnologica e ingenieril. Sin embargo, la operacion eficiente de esta red requiere una gestion de datos transaccionales altamente precisa. Este documento detalla el diseno de una base de datos relacional orientada a la gestion y el analisis de datos masivos generados por el sistema de transporte por cable, con un enfoque principal en el control de recaudaciones y la optima circulacion de pasajeros.

2. Problematica

La topografia accidentada y la severa congestion vehicular han consolidado a la Red "Mi Teleferico" como una arteria vital de transporte masivo. Sin embargo, administrar esta infraestructura requiere el procesamiento en tiempo real de miles de eventos diarios en los molinetes de acceso. El principal cuello de botella analitico radica en la falta de un diseno de base de datos optimizado que permita procesar volumenes masivos de informacion transaccional. Esto impide a los operadores identificar con precision los picos de saturacion por estacion y aplicar de manera automatizada y eficiente la logica tarifaria dinamica del sistema (cobro base de 3 Bs para la primera linea, cobro reducido de 2 Bs para transbordos, y validacion estricta de tarifas preferenciales).

3.2 Objetivos Especificos

  • Estructurar el registro transaccional de ingresos en los molinetes, capturando variables criticas (ID de medio de acceso, fecha, hora, estacion y linea) para mapear el volumen y la distribucion temporal de pasajeros.
  • Implementar la logica automatizada de cobro mediante tarifas dinamicas, aplicando los costos base, costos por transbordo y descuentos correspondientes a tarjetas preferenciales (adultos mayores, personas con discapacidad y estudiantes).
  • Clasificar los medios de acceso de los pasajeros, diferenciando el registro informatico entre tickets de un solo uso y tarjetas recargables (normales y preferenciales).

4. Requerimientos del Sistema

4.1 Requerimientos Funcionales (RF)

  • RF-01: El sistema debe registrar individualmente cada ingreso en los molinetes, capturando la marca de tiempo exacta, estacion de origen y deduciendo el saldo correspondiente de la tarjeta.
  • RF-02: El sistema debe permitir la emision y el registro de tickets fisicos de un solo uso (Normal, Turismo) generados en las ventanillas de cada estacion.
  • RF-03: El sistema debe calcular automaticamente el monto a cobrar validando si corresponde a una tarifa completa o a un descuento porcentual por tipo de tarjeta preferencial.
  • RF-04: El sistema debe aplicar automaticamente tarifas reducidas (Ej. 2 Bs) cuando detecte un transbordo entre lineas diferentes dentro del limite de tiempo establecido.
  • RF-05: El sistema debe registrar el historial de recargas de saldo, almacenando el monto, fecha, hora, estacion y metodo de pago utilizado.
  • RF-06: El sistema debe gestionar y diferenciar la emision de tarjetas segun su categoria (Normal, Preferencial para adultos mayores/discapacitados, Estudiantil) para la correcta aplicacion de reglas tarifarias.
  • RF-07: El sistema debe calcular el volumen total de pasajeros ingresados, filtrable de manera dinamica por estacion, linea y franja horaria.
  • RF-08: El sistema debe registrar y almacenar el historial detallado de viajes asociados a cada identificador unico de tarjeta.
  • RF-09: El sistema debe calcular la recaudacion total consolidando los ingresos por venta de tarjetas nuevas, recargas de saldo y emision de tickets en ventanilla.
  • RF-10: El sistema debe permitir a los usuarios consultar el saldo disponible de su tarjeta a traves de las terminales de molinetes sin generar un cargo transaccional.
